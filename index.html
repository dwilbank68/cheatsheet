Ruby Loop

Ruby has additional looping structures (apart from the common For..in and While loops) that are specific in use and aim to be a more semantic approach to a problem. Keep in mind that these can be replicated with the traditional For..in and While loops!

Ruby looping structures accept blocks as arguments. Blocks can be created using either curly brace notation ( ex: { |x| puts x } ) or do...end notation. Each ruby loop uses a block variable, which is denoted using pipes ( ex: |x| ) and represents the object being used during each iteration.

Each Loop
Example

Executes code once for each element in a collection.

# shorter route
array.each { |x| puts x}

# longer route
array.each do |x|
  puts x
end
Times Loop
Example

Executes code a given number of times. A Times Loop must be used with a variable containing a number or a number itself.

# shorter route
10.times { puts "hello world"}

# longer route
10.times do |x|
  puts x
end

Note that the first block excluded a local variable. If used, the local variable in the times loop will represent the number being used at a specific iteration in the loop. This number starts at zero. In the above example, the second times loop would print every number from 0-9.

Until Loop
Example

Executes code until a condition is true. This is the same as a negated while loop. ex: !(while)

# shorter route
# There is no short route

until false
  puts "infinite loop, yo"
end
Loop Loop
Example

Executes code in an infinite loop.

# shorter route
loop { puts "another infinite loop" }

# longer route 
loop do 
  puts "Infinite loop for days..."
end
